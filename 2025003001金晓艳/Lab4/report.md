# Lab4 实验报告

## 1. 应用界面结构说明
应用整体使用 Column 垂直布局，将界面分为上下两部分。上方显示骰子图片，下方放置 Roll 按钮。整体设置居中对齐，使图片和按钮在屏幕中水平、垂直都居中显示，结构清晰简洁。

## 2. 如何使用 Compose 状态保存骰子结果
使用 remember 配合 mutableStateOf 来保存骰子点数：
var result by remember { mutableStateOf(1) }
remember 保证状态在界面重组时不会丢失，mutableStateOf 使得 result 变化时可以触发界面刷新，从而保存并更新当前骰子点数。

## 3. 如何根据点数切换图片资源
使用 when 表达式根据 result 的值选择对应的图片资源。当 result 为 1~6 时，分别对应 dice_1 到 dice_6 图片。状态改变时，图片资源会自动更新，实现骰子图片切换。

## 4. 设置的断点及观察内容
在按钮点击事件中的 result = (1..6).random() 这一行设置断点。
主要观察：
1. 点击按钮后程序是否正常暂停
2. result 变量从旧值更新为新随机数的过程
3. 随机数生成是否在 1~6 范围内

## 5. Step Into、Step Over、Step Out 使用体会
Step Over 用于单步执行当前行，不进入函数内部，适合快速查看变量变化。
Step Into 可以进入函数内部查看执行细节，便于理解代码调用流程。
Step Out 能够快速跳出当前函数，回到上层调用位置，提高调试效率。

## 6. 遇到的问题与解决过程
1. 调试面板不显示变量
解决：改用调试模式启动，点击按钮触发断点后，变量面板正常显示。
2. 图片不随点击更新
解决：添加 remember 保存状态，使状态变化可以触发界面重组。
3. 调试时进入 Compose 源码
解决：这是正常重组流程，只需关注自身代码中的变量变化即可。

## 结论
按钮点击后图片可以自动刷新，是因为修改了被 Compose 状态管理的 result 变量，触发了界面重组。
调试器中变量的值与界面显示的骰子点数完全一致，说明逻辑正确，运行正常。
